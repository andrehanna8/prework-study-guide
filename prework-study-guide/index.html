HTML

    HTML stands for Hypertext Markup Language and is used to give webpage content a structure

    There are multiple elements or "tags" that are used to make up this structure such as:

  • The head element which contains information about the webpage.
  • The body element which contains information about the webpage.
  • The title element which defines the title for a webpage.
  • The style element which defines CSS styles associated with a website.
  • The base element which defines the base URL for the webpage.
  • The link element which connects an external resource to the HTML document.
  • The meta element which defines metadata such as the character set, description keywords, author and viewport.
  • The script element which embeds client-side scripts in the HTML document.
  • The viewport element which sets the default view for our webpage.

Git

    Here are some Git commands to start off:

  • ‘pwd’- displays current working directory
  • ‘ls’ - displays list of files and folders in the current directory
  • ‘cd’ - navigates between directories in the path
  • ‘cd ..” navigates backwards, one level at a time, through the path of your directories
  • ‘mkdir’ - creates a new directory
  • ‘touch’ - creates a file inside a directory
  • “start” - opens a file

    • Here is some important vocabulary:

  • “Repo” - every project on github is called this - a folder for your project that contains all of the files and their revision history
  • “Branch” - workspace where you can make changes that don't affect the live site. Create a new branch for your work, and name it to best describe work
  • “Markdown or (.md) - the way content files are formatted in github - A way to write for the web using plain text
  • “Commit changes” - saving a new version of a file you’re working on, adds changes to your branch
  • “Pull Request (PR) - asking for changes on a branch to merge into another branch. Way to discuss and review work with others
  • “Merge” - when a PR is approved and changes are “pulled in” or merged into main branch. Changes publish to live site
  • “Issue” - way to report new tasks or content fixes or track progress on project board. Open an issue for a new task and close once done
  • Feature Branch - Holds code that is currently being developed for a single feature of the application. Is also called a “working branch”
  • Develop Branch - Can hold the code for multiple feature branches that havent been merged into the main branch yet. Working branches can be kept up to date with the code in the main branch but the additions and changes in the working branches do not affect the final product until they are approved and reviewed.
